# OSAC Deployment on CRC

Simplified OSAC stack for local CRC testing.

## Architecture

```
cert-manager (cluster-wide)
  └─> trust-manager → CA bundle
  └─> ClusterIssuer (osac-ca, self-signed)
  └─> Certificates

CloudNativePG operator (postgres namespace)
  └─> PostgreSQL cluster (2 replicas)
      - osac-1, osac-2
      - keycloak DB (unused in simplified stack)
      - service DB (used by OSAC gRPC)

OSAC stack (osac namespace)
  ├─> osac-grpc (ghcr.io/osac-project/fulfillment-service:main)
  │   - Port: 8010
  │   - Connects to: osac-rw.postgres.svc.cluster.local:5432/service
  │   - Migrations: Handled cleanly by CloudNativePG
  └─> osac-rest (ghcr.io/osac-project/fulfillment-service:main)
      - Port: 8000
      - Proxies to osac-grpc:8010
```

## What's Different from Production (INSTALL.md)

**Simplified** (removed for CRC resource constraints):
- ❌ Keycloak (2 pods) - using unauthenticated mode for testing
- ❌ Authorino operator - no external auth enforcement
- ❌ Controller component - not needed for basic API testing
- ❌ Python OIDC server - not needed without controller

**Kept**:
- ✅ cert-manager + trust-manager
- ✅ CloudNativePG operator (production-grade DB)
- ✅ Self-signed CA infrastructure
- ✅ gRPC + REST gateway

## Services

| Component | Namespace | Service | Port |
|-----------|-----------|---------|------|
| PostgreSQL (primary) | postgres | osac-rw | 5432 |
| PostgreSQL (read replicas) | postgres | osac-r, osac-ro | 5432 |
| OSAC gRPC | osac | osac-grpc | 8010 |
| OSAC REST | osac | osac-rest | 8000, 8012 (metrics) |

## Database Connection

The gRPC server connects to the CNPG primary via:
```
postgres://service:<password>@osac-rw.postgres.svc.cluster.local:5432/service?sslmode=require
```

Password is stored in `postgres/osac-service-credentials` secret (auto-generated by CNPG).

## Deployment Steps (from scratch)

See commit history for full deployment. Key steps:

1. Install cert-manager + trust-manager via Helm
2. Create self-signed CA + ClusterIssuer
3. Install CloudNativePG operator
4. Create PostgreSQL cluster with keycloak + service databases
5. Deploy OSAC gRPC + REST from manifests

## Known Issues

- OIDC server crashes (missing cryptography dependency in python:3.12-slim)
  - Not needed for unauthenticated testing
  - For production, use full Keycloak deployment

- REST API /health endpoint returns 404
  - Expected - OSAC uses different health check paths
  - Services are healthy (check logs show "Start serving")

## Migration Bug (Avoided)

Manual K8s manifests with plain postgres:16 image hit go-migrate "dirty" bug
where every migration marks itself dirty before running. **CloudNativePG avoids
this** - migrations ran cleanly on first boot.
